# 字段语义与陷阱

本文档解释 fund-scout 输出数据中每个字段的语义、单位、数据来源、典型缺失场景和踩坑点。每次抓出诡异字段时优先来这里查。

## 基础字段

### `code`

- 6位数字基金代码
- C 类基金代码 ≠ 主代码（A 类）。例：易方达全球成长精选 A=012920，C=012922
- CSRC 系统认 A 类主代码，搜 C 类常常搜不到 → 必须有 `main_code` 字段配合
- **来源**：用户输入 / 搜索接口

### `name`

- 全名，含份额类型后缀如「(QDII)C」「(LOF)」「A」「人民币」「美元」「H」
- 同一只基金不同份额可能仅在最后一个字符不同 → 用 substring 匹配易出错
- **来源**：基金主页 `<title>` 标签

### `short_name`

- 基金简称，不含份额类型后缀
- 用于 CSRC 搜索的备用关键词
- **来源**：搜索接口 `fundcode_search.js`

### `type`

- 取值：「股票指数」「股票型」「混合型」「债券型」「QDII」「FOF」「货币型」
- 主页 type 字段不一定准（有时显示为「指数型」），档案页 `jbgk` 更可信
- **来源**：基金主页 infoOfFund 块

### `risk`

- 风险等级，如「中高风险」「低风险」
- **来源**：基金主页 infoOfFund 块

### `scale`

- 单位：亿元
- 主页 vs 档案页可能有 1~2 周延迟差距，**正常 < 5%；> 10% 触发 cross-source 告警**
- C 类 / E 类 / 美元份额各自独立计算规模
- 「资产净值」 vs 「资产规模」：前者 = 后者 - 负债，差异通常 < 1%
- **来源**：基金主页（`规模：XX亿元`）+ 档案页（`资产规模：XX亿`），档案页更稳

### `nav`、`nav_date`

- nav = 单位净值，按交易日更新
- nav_date 主页只有 MM-DD（无年份）；要拿带年份的，去 `lsjz` API
- QDII 基金有 T+1 / T+2 净值滞后，nav_date 比国内基金落后
- **来源**：基金主页（实时）+ NAV 历史 API（带完整日期）

### `company`

- 基金管理人（基金公司全称）
- **来源**：基金主页 infoOfFund 块

### `found_date`

- 基金成立日期，格式 YYYY-MM-DD
- **来源**：基金主页 infoOfFund 块

### `benchmark`

- 跟踪标的 / 业绩比较基准
- 用 strip_tags 清洗 HTML 后截取前 60 字符
- **来源**：档案页 `jbgk`

## 收益率字段

| 字段 | 含义 | 来源 | 注意点 |
|------|------|------|--------|
| `return_1w` | 近 1 周 | 排行 | 单位：% |
| `return_1m` | 近 1 月 | 主页 / 排行 | 单位：% |
| `return_3m` | 近 3 月 | 主页 / 排行 | 单位：% |
| `return_6m` | 近 6 月 | 主页 / 排行 | 单位：% |
| `return_1y` | 近 1 年 | 主页 / 排行 | 最常用 |
| `return_3y` | 近 3 年 | 主页 / 排行 | 累计回报，非年化 |
| `return_ytd` | 今年以来 | 排行 | 主页有时不显示 |
| `return_since_inception` | 成立以来 | 排行 | 累计回报 |
| `return_sl` | 同 since（旧别名） | 主页 | 向后兼容 |

⚠ 排行接口与主页可能差 0.1~1pp（净值更新时间不同），允许 ≤ 1pp 差异。差异 > 1pp 应触发告警。

## 申购状态字段

| 字段 | 类型 | 取值 |
|------|------|------|
| `purchase_status` | str | `开放` / `限大额` / `限小额` / `暂停` / `未知` / `暂停(限大额)` / `暂停(限小额)` |
| `purchase_limit` | str | `无限制` / `X万` / `X元` / `0` |
| `effectively_closed` | bool | True 表示实际等同暂停 |

### purchase_status 四态详解

| 显示文本 | `purchase_status` | `purchase_limit` | 解读 |
|---------|-------------------|------------------|------|
| `开放申购` | `"开放"` | `"无限制"` | 正常可买 |
| `限大额(单日累计购买上限 X 万元)` | `"限大额"` | `"X万"` | 大额限购，小额可买 |
| `限大额(单日累计购买上限 X 元)` | `"限小额"` | `"X元"` | **实际接近暂停申购** |
| `暂停申购` | `"暂停"` | `"0"` | 完全不能买 |

### 易混点

- `限小额` ≠ 「小额限购」。是「限大额(单日上限 X 元)」即只准小额买入但实际门槛极低，本质是关闭状态。
- 当限额是**元**而非**万元**，且金额 ≤ 1000 元时，附加 `effectively_closed: true`。**禁止**把这种基金宣传为「可申购」。
- `暂停申购` 还分「全部暂停」和「仅大额暂停」，主页 staticCell 通常合并显示，校验时看具体文本。
- **来源**：基金主页 HTML 解析（`_parse_purchase_status` 方法）

## 费率字段

| 字段 | 含义 | 单位 | 来源 | 说明 |
|------|------|------|------|------|
| `mgmt_fee` | 管理费率 | %/年 | 档案页 | 几乎所有基金都有 |
| `custody_fee` | 托管费率 | %/年 | 档案页 | 几乎所有基金都有 |
| `service_fee` | 销售服务费率 | %/年 | 档案页 | **C 类基金特有**，A 类一般为 0 |
| `total_fee` | 综合年费率 | %/年 | 计算 | = mgmt + custody + service |

### total_fee 计算规则

```python
fees = [mgmt_fee, custody_fee, service_fee]
if any(f > 0 for f in fees):
    total_fee = round(sum(fees), 4)
```

- 如果三个费率字段都为 0 或 None，则不计算 total_fee
- 保留 4 位小数，与 `validate.py` 中 `abs(expected - total) > 0.02` 的容差匹配

### C 类 vs A 类费率差异

| 费率项 | A 类 | C 类 |
|--------|------|------|
| 管理费 mgmt_fee | 有（通常 0.5~1.5%） | 有（与 A 类相同） |
| 托管费 custody_fee | 有（通常 0.1~0.25%） | 有（与 A 类相同） |
| 销售服务费 service_fee | **通常为 0** | **必有**（通常 0.2~0.6%） |
| 申购费 sub_fee | 有（通常 0.12~1.5%） | **无**（0 申购费） |
| 赎回费 red_fee | 持有 < 7 天 1.5% | 持有 < 7 天 1.5% |
| **综合费 total_fee** | ≈ mgmt + custody | ≈ mgmt + custody + service |

> C 类虽然 0 申购费，但每年多收 service_fee，长期持有总成本反而高于 A 类。短期持有 C 类更划算。

## 持仓字段

```jsonc
{
  "top10_holdings": [
    {
      "code": "00700",       // 港股5位 / A股6位 / 美股带字母
      "name": "腾讯控股",
      "pct": "8.50%",        // 占基金净值百分比
      "shares": "100,000",   // 持股数（注意千分位逗号）
      "value": "8,500,000.00" // 持仓市值（元）
    }
  ]
}
```

### 已知坑

1. **港股代码不是纯数字**（如 `00700`），但部分接口返回 `'00700'` 字符串可以用 isdigit() 通过；少数返回带字母的（如 `BABA-SW`）会被误过滤。
2. **季度去重**：year=2026 接口和 year=2025 接口都可能返回 `2025年4季度`，需按 quarter 文本去重。
3. **占比解析**：百分号有半角 `%` 和全角 `％` 两种。
4. **持股数千分位**：可能是 `,` 也可能是中文 `，`。
5. **NA 占位**：偶尔返回 `--` 或 `-`，应解析为 0 或 null。
6. **来源**：天天基金持仓接口 `FundArchivesDatas.aspx`

## 经理字段

```jsonc
{
  "tenures": [
    {
      "start": "2024-03-15",
      "end": "至今",
      "period": "2024-03-15 ~ 至今",
      "managers": ["张三", "李四"],
      "days": "428",
      "return": "42.18%"
    }
  ],
  "current_tenure": { /* end == "至今" 的那条 */ },
  "manager_profiles": [
    {
      "name": "张三",
      "avatar": "https://j5.dfcfw.com/...jpg",
      "appoint_date": "2024-03-15",
      "bio": "经济学硕士，2018年加入..."
    }
  ]
}
```

### 注意

- `current_tenure` 可能是 None（基金清盘 / 数据延迟）
- `avatar` 如果以 `//` 开头，要补 `https:` 前缀
- 多基金经理：一只基金可能 2~3 人共管，列表都要拿
- 历任 vs 当前：用 `end == "至今"` 判定，不要用日期算
- **来源**：经理页 `jjjl_{code}.html`（唯一可靠源）

## 市场分布字段

```jsonc
{
  "market_distribution": {
    "美国": 92.50,
    "中国香港": 4.50,
    "中国内地": 1.20,
    "_source": "csrc_2026Q1",
    "_total_pct": 98.20,
    "_inferred": false
  }
}
```

### market_distribution 来源差异

| 来源 | 准确性 | 说明 |
|------|--------|------|
| CSRC 季报 PDF | **高** | 法定披露原文，按持仓股票注册地统计 |
| 天天基金「区域分布」 | **低** | 根据持仓股名做粗暴推断，错误率高 |

> **禁止**使用天天基金的「区域分布」作为市场分布数据源。涉及市场分布必须走 CSRC 季报 PDF。

### 注意

- 数值是占基金净值百分比，**国家合计通常 60~95%**（其余是现金、债券、未投出资金）
- 如果合计 > 100% → 一定是解析错误（可能把"小计"当成数据行了）
- 「中国香港」 vs 「香港」要统一（CSRC 不同基金披露用词不同），代码中通过 `COUNTRY_ALIAS` 映射
- 联接 / FOF 基金没有这张表 → 必须 `_inferred: true`，不要假装抓到了
- `_source` 标注来源季度，格式 `csrc_YYYYQN`
- `_inferred` 为 true 时表示数据缺失，不可用于分析

### 国家名规范

代码中 `COUNTRY_ALIAS` 统一映射：

```python
COUNTRY_ALIAS = {
    "香港": "中国香港",
    "台湾": "中国台湾",
}
```

已知国家名集合：

```
美国、中国内地、中国香港、日本、韩国、英国、德国、法国、印度、新加坡、
澳大利亚、加拿大、瑞士、荷兰、巴西、以色列、开曼群岛、百慕大、中国台湾、
意大利、西班牙、墨西哥、南非、泰国、印度尼西亚、马来西亚、越南
```

## 回撤字段

| 字段 | 含义 | 计算方式 | 来源 |
|------|------|---------|------|
| `drawdown_1y` | 近1年最大回撤 | 从近1年NAV序列中计算峰值到谷底最大跌幅 | NAV API + 计算 |
| `drawdown_3y` | 近3年最大回撤 | 从近3年NAV序列中计算峰值到谷底最大跌幅 | NAV API + 计算 |

### 计算逻辑

1. 从 NAV API 分页获取全量历史净值
2. 按日期升序排列
3. 遍历计算：维护历史峰值 peak，当前回撤 = (peak - nav) / peak
4. 取最大回撤值

### 陷阱

- **必须分页拿全量**：单页只有 20 条，不拿全会严重低估回撤
- NAV 数据 < 20 条时回撤不可靠 → 触发 `drawdown_insufficient_navs` 警告
- QDII 基金 T+1/T+2 净值滞后，回撤计算日期边界需注意

## Warnings 字段

`_warnings` 列表里出现的常见标记：

| 标记 | 含义 | 是否致命 |
|------|------|---------|
| `{code}_main_page_failed` | 主页抓取失败 | 是 |
| `{code}_manager_page_failed` | 经理页失败 | 否（影响可视化卡片） |
| `{code}_holdings_empty` | 持仓接口返回空 | 否 |
| `{code}_no_rank_data` | 排行接口未覆盖 | 否（影响 return 字段） |
| `{code}_missing_xxx,yyy` | 字段缺失列表 | 看具体字段 |
| `{code}_csrc_not_found` | CSRC 搜不到 instance id | 否 |
| `{code}_pdf_parse_failed` | PDF 解析失败 | 否（市场分布字段缺失） |
| `{code}_drawdown_insufficient_navs` | NAV 数据 < 20 条 | 否 |
| `{code}_fee_mismatch` | total_fee 与 mgmt+custody+service 不等 | 是 |
| `{code}_fetch_failed` | 整个抓取流程异常 | 是 |

## 衍生字段

由 `fetcher.py` 的调度逻辑组合计算，必须基于真实抓取数据：

| 字段 | 含义 | 计算 |
|------|------|------|
| `update_date` | 数据更新日期 | `time.strftime("%Y-%m-%d")` |
| `count` | 基金数量 | `len(funds)` |
| `_validation` | 校验结果 | 由 `validate.py` 生成 |
| `_warnings` | 警告列表 | 由 `validate.py` 生成 |

⚠ **禁止**在没有真实抓取数据的情况下凭空生成任何衍生字段。
