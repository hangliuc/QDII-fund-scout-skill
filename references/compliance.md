# 合规指南

本文档列出 QDII-fund-scout 使用过程中必须遵守的合规红线、数据源使用规范、免责声明和使用者责任。

---

## 1. 数据源合规红线

本工具涉及三个数据源，每种数据有不同的合规要求：

### 1.1 天天基金（fund.eastmoney.com）

| 事项 | 说明 |
|------|------|
| 数据类型 | 净值、收益率、费率、持仓、基金经理等信息页面 |
| robots.txt | 未公开限制路径，但按行业惯例需限速 |
| 强制约束 | 请求间隔 ≥ 0.5s，禁止并发，禁止绕过反爬 |
| 可接受使用 | 个人/少量基金（≤ 50 只/天）的日常查询 |
| 高风险使用 | 全量遍历、高频轮询、多实例并行、代理 IP 池绕过限速 |

> **警告**：天天基金有成熟的反爬机制。高频访问会导致 IP 被封，封禁时间从 5 分钟到永久不等。多人共享同一出口 IP（如公司 NAT）时风险叠加。

### 1.2 好买基金（www.howbuy.com）

| 事项 | 说明 |
|------|------|
| 角色 | 备用数据源，仅在天天基金不可用时自动切换 |
| 强制约束 | 请求间隔 ≥ 0.5s，禁止并发 |
| 注意 | 好买基金同样有反爬机制，不可作为主数据源高频抓取 |

### 1.3 证监会季报（eid.csrc.gov.cn）

| 事项 | 说明 |
|------|------|
| 数据类型 | 基金季度报告的 PDF 文件（法定披露文件） |
| 法律属性 | 政府公开信息，依法可查阅 |
| 强制约束 | 请求间隔 ≥ 1.0s，PDF 下载间隔 ≥ 1.0s |
| PDF 再分发 | **禁止**直接分发/转载 PDF 原文 |
| 季度持仓 | 从 PDF 中提取的结构化数据（行业/国家分布）属于衍生数据，可合理使用 |

> **重要**：证监会季报的 PDF 原文版权归基金管理人所有，本工具仅做 PDF 解析提取结构化数据，**不提供 PDF 原文的存储和再分发**。提取结果属于衍生数据，但使用者仍需注明数据来源。

## 2. 数据使用红线

### 2.1 禁止行为

| 红线 | 说明 | 风险等级 |
|------|------|---------|
| 商业转售 | 将抓取的结构化数据、PDF 提取内容以任何形式收费转售 | ⚠️ 高 |
| 竞争性产品 | 构建与天天基金、好买基金等平台功能类似且有竞争关系的产品 | ⚠️ 高 |
| 付费 API 化 | 将本工具包装为付费 API/数据服务提供给第三方 | ⚠️ 高 |
| 大规模分发 | 将工具输出数据推送到大规模公开群组、网站或 App | ⚠️ 中 |
| 投资建议 | 基于工具数据向他人提供基金买卖建议 | ⚠️ 中 |
| 伪造数据 | 用大模型「常识」补全未抓取到的字段 | ⚠️ 中 |
| PDF 原文分发 | 将季报 PDF 文件直接复制、分发或公开托管 | ⚠️ 中 |
| 高频轮询 | 以低于 1 分钟的间隔反复查询同一只基金 | ⚠️ 低（但会被封） |

### 2.2 允许行为

| 行为 | 说明 |
|------|------|
| 个人持仓查询 | 查询自己持有的几只基金数据 |
| 小群分享 | 推送到个人或小规模好友群（< 50 人）|
| 定时快照 | 每日定时查询并推送，用于跟踪自己关注的基金 |
| 数据分析 | 对结构化数据进行个人维度的统计分析 |
| 学习研究 | 研究 QDII 基金的限购/费率/收益模式 |

### 2.3 规模化运营的特殊风险

随着用户量和调用量增长，以下风险会逐步放大：

| 风险 | 触发条件 | 后果 |
|------|---------|------|
| 数据源封禁 IP | 日均请求量超过来源站阈值 | 工具完全不可用 |
| 数据源法律函件 | 被监测到批量获取结构化数据 | 要求停止访问 |
| 证监会/版权方投诉 | PDF 解析内容被认为侵犯版权 | 下架要求 |
| 用户数据误判 | 数据延迟/错误导致用户做出错误投资决策 | 追责风险 |
| 合规审查 | 被监管部门注意到存在规模化数据分发行为 | 行政/刑事责任 |

**规模化运营建议**：

1. **部署反代缓存**：同一只基金的数据在 1 小时内不重复请求源站，降低源站压力
2. **用户分级限速**：总并发请求数 ≤ 1，总 QPS ≤ 1
3. **日志存证**：保留数据获取时间戳，证明数据来源和时效性
4. **及时整改**：如收到数据源平台停止要求，应在 48 小时内响应
5. **法律咨询**：在用户量达到 1000+ 后，建议咨询专业律师评估合规性

## 3. robots.txt 合规说明

### 3.1 天天基金

天天基金未公开 `robots.txt`，但基于行业惯例：

- 公开信息页面（基金主页 `/fund/{code}.html`、档案页 `/jbgk_{code}.html`、经理页 `/jjjl_{code}.html`）可访问
- API 接口（`/f10/lsjz` 等）属于平台内部接口，限速使用
- 不得高频访问导致平台服务受影响

### 3.2 证监会

证监会网站为政府信息披露平台：

- 季报搜索和 PDF 下载为公开服务
- 不得对政府网站造成过大访问压力（建议 ≥ 2s 间隔）

### 3.3 好买基金

好买基金有活跃的反爬机制：

- 建议仅在天天基金不可用时作为备用源使用
- 不低于 1.0s 请求间隔

## 5. 数据源使用规范

### 5.1 强制限速

| 规则 | 参数 | 说明 |
|------|------|------|
| 最小请求间隔 | 0.5s | `rate_limit` 最小值，不可关闭 |
| 推荐请求间隔 | 1.0~2.0s | 关键接口（如 NAV 历史、PDF 下载）建议更大间隔 |
| 随机抖动 | 0.5~2.0s | 每次请求随机 sleep 避免规律性请求 |
| 连续失败熔断 | 30 次 | 连续 30 次失败自动停止并报错 |
| 禁止并发 | — | 所有请求必须串行，禁止多线程/协程/异步并发 |
| 单 IP 限额 | ~50 次/会话 | 天天基金约 50 次请求后可能被封 5~10 分钟 |

### 5.2 禁止绕过反爬

- 禁止使用代理池轮换 IP 以规避限速
- 禁止伪造 Cookie/Session 绕过登录验证
- 禁止使用无头浏览器自动化（Selenium/Playwright）绕过 JS 渲染
- 禁止逆向工程加密接口参数
- 禁止绕过 `robots.txt` 中的 Disallow 规则

### 5.3 合规原则

1. **最小必要**：只抓取所需数据，不批量遍历全站
2. **限速至上**：遵守最小请求间隔，不并发、不绕过
3. **用途受限**：仅用于个人学习研究，禁商业转售
4. **尊重平台**：如平台要求停止，应立即响应
5. **标注来源**：所有输出内容标注数据来源
6. **合理缓存**：同一基金数据 1 小时内不重复请求
